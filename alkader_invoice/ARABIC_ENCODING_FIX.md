# Arabic Encoding Fix - Complete Solution

## Problem
Arabic text appears as garbled characters (e.g., "ÙØ§ØªÙˆØ±Ø©" instead of "فاتورة ضريبية") in PDF invoices generated by Odoo.

## Root Cause
The issue occurs when:
1. Arabic fonts are not installed on the Odoo server
2. CSS classes override global fonts with non-Arabic fonts
3. wkhtmltopdf doesn't receive proper UTF-8 encoding configuration
4. The HTML template lacks proper encoding declarations

## Solutions Applied

### 1. Updated All CSS Font Declarations
All CSS classes now use Arabic-compatible fonts:
- `h1`, `h2`, `h3`, `h4`
- `.s1` through `.s12`
- `.th4`, `.th5`
- `p` and other elements

Font fallback chain:
```css
'KacstOne', 'KacstFarsi', 'DejaVu Sans', 'Arial Unicode MS', 'Noto Sans Arabic', 'Noto Kufi Arabic', sans-serif
```

### 2. Added UTF-8 Encoding Declarations
- Added `@charset "UTF-8"` at the start of CSS
- Added `<meta charset="utf-8"/>` in HTML head
- Added `<meta http-equiv="Content-Type" content="text/html; charset=utf-8"/>` in multiple locations
- Set HTML lang attribute to "ar"

### 3. Configured Report Action
Updated `reports/report.xml` to include encoding options:
```xml
<field name="options" eval="{'encoding': 'UTF-8', 'enable-local-file-access': True}"/>
```

### 4. Enhanced Report Rendering
Added `_render_qweb_html` method in `invoice_report.py` to ensure proper UTF-8 encoding during rendering.

## Required Server Configuration

### Step 1: Install Arabic Fonts
```bash
# Update package list
sudo apt-get update

# Install Arabic font packages
sudo apt-get install -y fonts-arabeyes fonts-farsiweb fonts-kacst fonts-kacst-one
sudo apt-get install -y fonts-noto-naskh-arabic fonts-noto-kufi-arabic

# Alternative: Install comprehensive Arabic fonts
sudo apt-get install -y fonts-noto-arabic fonts-noto-arabic-ui

# Refresh font cache
fc-cache -f -v

# Verify fonts are installed
fc-list :lang=ar | head -20
```

### Step 2: Verify wkhtmltopdf Installation
```bash
# Check wkhtmltopdf version
wkhtmltopdf --version

# Should show version 0.12.5 or higher
# If not installed or wrong version:
sudo apt-get install -y wkhtmltopdf

# Or install patched version for better Arabic support:
wget https://github.com/wkhtmltopdf/packaging/releases/download/0.12.6.1-2/wkhtmltox_0.12.6.1-2.jammy_amd64.deb
sudo dpkg -i wkhtmltox_0.12.6.1-2.jammy_amd64.deb
sudo apt-get -f install
```

### Step 3: Configure Odoo
Update your `odoo.conf` file:
```ini
[options]
# Ensure UTF-8 encoding
reportgz = False
```

### Step 4: Restart Odoo
```bash
# If using systemd
sudo systemctl restart odoo

# Or if running manually, restart your Odoo process
```

### Step 5: Update Module
1. Go to Apps in Odoo
2. Find "Alkader Invoice" module
3. Click "Upgrade"
4. Or restart Odoo server to reload the module

## Troubleshooting

### If Arabic still shows as garbled:

1. **Check Odoo Logs**
   ```bash
   tail -f /var/log/odoo/odoo-server.log | grep -i "font\|encoding\|utf"
   ```

2. **Verify XML File Encoding**
   Ensure `report_custom_invoice.xml` is saved as UTF-8:
   ```bash
   file -bi reports/report_custom_invoice.xml
   # Should show: text/xml; charset=utf-8
   ```

3. **Check Browser Encoding**
   - Clear browser cache
   - Ensure browser is set to UTF-8
   - Try a different browser

4. **Verify Database Encoding**
   ```sql
   SHOW VARIABLES LIKE 'character_set%';
   -- Should show utf8mb4 or utf8
   ```

5. **Test with Simple Template**
   Create a minimal test template with just Arabic text to isolate the issue.

- All hardcoded Arabic text in the template should display correctly once fonts are installed
- Dynamic Arabic text from database fields should also work with proper encoding
- If using custom fonts, ensure they're installed system-wide, not just in Odoo

## Support

If issues persist after following all steps:
1. Check Odoo version compatibility (tested on Odoo 19)
2. Verify wkhtmltopdf version and patches
3. Check system locale settings: `locale -a | grep -i utf`
4. Ensure Python encoding: `python3 -c "import sys; print(sys.getdefaultencoding())"` should show `utf-8`
